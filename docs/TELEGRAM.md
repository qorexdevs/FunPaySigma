# Telegram-панель FunPay Sigma

Документ описывает Telegram-слой Sigma 2.15. Панель рассчитана на владельца
FunPay-аккаунта и не заменяет `FunPayAPI`: Telegram-обработчики вызывают тот же
`Account`, что и основной Runner.

## Что появилось в 2.14

| Экран | Возможности |
| --- | --- |
| `/dashboard` (`/status`) | Аптайм, версия, состояние FunPay-сессии, доступный баланс, активные заказы, лоты, автовыдача, плагины, прокси и нагрузка процесса. |
| `/sales` (`/orders`) | Последние восемь продаж с покупателем, статусом, суммой, датой и ссылкой на заказ. Кнопка `copy_text` используется при наличии в установленной Telegram-библиотеке; на старых версиях ID показывается через callback-alert. |
| `/lot_health` (`/lot_status`) | Сверка кэшированного списка лотов с секциями автовыдачи и файлами складов. Кнопка обновления запрашивает профиль FunPay. |
| `/notification_digest` (`/digest`) | Количество включённых уведомлений, подключённых чатов, неподтверждённых заказов и заказов без отзыва. |
| `/telegram_proxy` | Отдельный HTTP(S)/SOCKS5/SOCKS5H-прокси для Telegram Bot API с проверкой через `getMe`. |

Все экраны открываются из страницы **Инструменты** меню. Навигация редактирует
текущее сообщение, а callback подтверждается сразу, поэтому повторные нажатия
не создают очередь «часиков» и не засоряют чат.

## Дерево меню

```text
/menu
├── Основное
├── Коммуникации
└── Инструменты
    ├── Графики / синхронизация / тикеты
    ├── Конфиги / прокси / авторизованные пользователи
    ├── Дашборд
    ├── Продажи
    ├── Здоровье лотов
    ├── Дайджест уведомлений
    └── Telegram-прокси
```

## Совместимость с Telegram

Sigma использует `pytelegrambotapi` и поддерживает безопасный fallback для
устаревших версий библиотеки. Новые элементы опираются на стабильные Bot API:

- inline keyboards и редактирование сообщений;
- `answerCallbackQuery` для мгновенного подтверждения действий;
- URL-кнопки заказов;
- `copy_text` buttons, если их предоставляет установленная библиотека.

Rich Messages, guest mode и managed bots из свежих changelog Bot API не
включены автоматически: они не нужны для администрирования FunPay и могут быть
недоступны в закреплённой версии `pytelegrambotapi`. Это оставляет панель
совместимой с уже установленными конфигурациями.

На 23 июля 2026 года официальный changelog указывает Bot API 10.2: Rich
Messages, ephemeral messages, Communities и `BotSubscriptionUpdated`. Для
Sigma практически применимы inline-навигация, редактирование сообщений и
`copy_text`; остальные возможности требуют отдельного пользовательского
сценария и потому не добавляются номинально.

Официальные материалы Telegram:

- [Bot API changelog](https://core.telegram.org/bots/api-changelog)
- [Bot API reference](https://core.telegram.org/bots/api)
- [Inline keyboards and buttons](https://core.telegram.org/bots/features#inline-keyboards)
- [Mini Apps](https://core.telegram.org/bots/webapps)

## Безопасность

Доступ к панели получают только пользователи из `storage/cache/tg_authorized_users.json`.
Дашборд не показывает `golden_key`, cookies, токены или содержимое складов.
Команды `/restart`, `/power_off`, `/update`, `/withdraw` и загрузка файлов
остаются отдельными административными действиями и не выполняются из новых
экранов автоматически.

При добавлении нового callback используйте строковый namespace `sigma.*` в
`tg_bot/CBT.py`. Старые числовые callback значения не переиспользуйте: их могут
содержать уже отправленные Telegram-сообщения и сторонние плагины Cardinal.

## Telegram-прокси при блокировке API

Параметр `[Telegram] proxy` не связан с секцией `[Proxy]`, которая обслуживает
FunPay. При запуске значение применяется к `telebot.apihelper.proxy` до создания
экземпляра `TeleBot`, поэтому через него проходят long polling, callback-ответы,
уведомления и запросы дополнительных Telegram-ботов.

Доступные способы настройки:

1. `/telegram_proxy` в работающей панели;
2. `SetupTelegramProxy.bat` на Windows;
3. `python setup_telegram_proxy.py` из корня проекта;
4. ввод прокси во время первой настройки до проверки токена BotFather.

Поддерживаются форматы:

```text
http://login:password@127.0.0.1:8080
socks5://127.0.0.1:1080
socks5h://login:password@127.0.0.1:1080
```

Перед сохранением выполняется запрос `getMe` именно через выбранный прокси.
В интерфейсе пароль маскируется. В `_main.cfg` значение сохраняется через тот
же механизм обфускации, что и Telegram token.
