# История изменений FunPay Sigma

## [2.14.1] — 23.07.2026

### Исправлено

- Апдейтер больше не зависит только от `api.github.com`: добавлены fallback через GitHub Atom feed и redirect страницы последнего релиза.
- При временной недоступности GitHub используется последний валидный release-кэш, включая просроченный, вместо немедленной ошибки.
- Архив обновления скачивается через обычный GitHub archive URL с резервным `codeload.github.com`.
- Загруженный ZIP проверяется до установки; пустой или повреждённый архив отклоняется.
- Время жизни свежего кэша сокращено до пяти минут, чтобы ручная проверка быстрее видела новые релизы.

### Проверено

- GitHub API rate limit → Atom fallback.
- Отказ API и Atom → release redirect fallback.
- Полная сетевая недоступность → stale-cache fallback.
- Формирование резервных GitHub/codeload URL для архива.

## [2.14.0] — 23.07.2026

### Добавлено

- Операционный Telegram-дашборд: аптайм, версия, FunPay-сессия, доступный баланс, активные заказы, лоты, автовыдача, плагины, прокси и нагрузка процесса.
- Экран последних продаж FunPay с быстрыми ссылками на заказы, статусами, ценой, датой и копированием ID.
- Проверка здоровья лотов: активность, покрытие автовыдачей, склады и лишние секции конфигурации.
- Дайджест Telegram-уведомлений и очередей подтверждения/отзывов.
- RU/EN/UK локализация новых команд и кнопок.
- Документация [Telegram-панели](docs/TELEGRAM.md), обновлённые README и FunPayAPI reference.

### Совместимость

- Новые callback-значения вынесены в namespace `sigma.*`; старые числовые значения Cardinal не изменены.
- Кнопка копирования использует Telegram `copy_text` при поддержке установленной библиотеки и callback fallback в старых версиях `pytelegrambotapi`.
- Runtime-каталоги, cookies, токены, конфиги и складские файлы не входят в изменения.
